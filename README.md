# Data Mining Lab Repository

This repository contains notebook-based lab work and datasets of my Data Mining Lab course. Each notebook captures a lab exercise or experiment workflow, including preprocessing, model building, and evaluation.

## Repository Contents

### Notebooks

- `Heart_failure.ipynb`  
  Heart failure prediction workflow using cleaned clinical data.

- `ML_Lab1.ipynb`  
  Lab 1 notebook: introductory data exploration and preprocessing foundations.

- `ML_Lab2.ipynb`  
  Lab 2 notebook: feature handling and core machine learning workflow.

- `Ml_Lab3.ipynb`  
  Lab 3 notebook: data preparation, experiments, and interpretation.

- `ML_Lab4.ipynb`  
  Lab 4 notebook: applied preprocessing, modeling, and observations.

- `ML_Lab5.ipynb`  
  Lab 5 notebook: model execution and comparative analysis.

- `ML_lab6.ipynb`  
  Lab 6 notebook: machine learning techniques and evaluation metrics.

- `ML_lab7_and_8.ipynb`  
  Combined Lab 7 and 8 notebook with end-to-end analysis and conclusions.

### Datasets

- `A Survey of Child Malnutrition Based on Machine Learning.csv`
- `cse.csv`
- `heart_2020_cleaned.csv`
- `Study_Hours.csv`

## Requirements

Recommended Python version:

- Python 3.9+

Typical packages used in data mining labs:

- pandas
- numpy
- matplotlib
- seaborn
- scikit-learn
- jupyter

Install packages with:

```bash
pip install pandas numpy matplotlib seaborn scikit-learn jupyter
```

## How to Run

1. Open this folder in VS Code.
2. Open any notebook (`.ipynb`) file.
3. Select a Python kernel/environment.
4. Run cells in order from top to bottom.

## Suggested Workflow

1. Start with `ML_Lab1.ipynb` and proceed sequentially.
2. Keep datasets in the repository root so paths continue to work.
3. For final review, revisit `ML_lab7_and_8.ipynb` and `Heart_failure.ipynb` for complete workflows.

## Notes

- Notebook outputs are intentionally preserved for documentation and result tracking.
- Some notebooks may rely on relative paths to datasets in the root folder.
